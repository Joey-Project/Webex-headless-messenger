# Thin CLI

## Summary
- Added `webex-headless` as a thin binary wrapper over the existing library API.
- Covered Device Grant authorization, token-file loading and refresh, `me`, room
  list/get/resolve, message list/direct/get/send/reply/delete, message polling,
  and a loopback sidecar receiver.
- Kept command parsing dependency-free for this first CLI slice; future command
  growth can migrate to a CLI framework without changing the library API.
- Documented the CLI auth and common room/message workflows in README.

## Current State
- One-shot REST commands emit pretty JSON for operator readability.
- Polling and sidecar receive commands emit compact JSON Lines for automation.
- Token files use `TokenSet` JSON; with client credentials present, the CLI uses
  `RefreshingTokenProvider<FileTokenStore>` and writes refreshed tokens back to
  disk.
- Sidecar receive defaults to loopback bind and requires a bearer token unless
  explicitly run with `--allow-unauthenticated`.

## Next Steps
- Keep richer CLI ergonomics, shell completions, and builder-style command
  abstractions deferred until command surface growth justifies them.

## Evidence
- Local validation: `cargo fmt --check`, `cargo test --all-features --all-targets`,
  `cargo clippy --all-features --all-targets -- -D warnings`,
  `cargo doc --no-deps --all-features`, `cargo run --bin webex-headless -- --help`,
  project journal validation, CLI sidecar mock E2E with the JS sidecar demo
  forwarding one `messages.created` event to `sidecar receive --max-events 1`,
  and read-only live `me` smoke with `.codex-tmp/webex-smoke/token.json`.
